# 天翼视联点位巡检系统首版基线（精简版）

## 1. 项目定位
首版定位为“视频可视巡检与故障派单系统”。

首版唯一核心目标：
- 让用户一眼看出哪些设备点位当前看不到视频画面
- 让这些异常点位立即进入派单与恢复闭环

## 2. 首版业务主线
固定主线：
1. 点位清单同步与展示
2. 视频巡检
3. 异常归类
4. 故障入池
5. 企业微信通知 / 派单
6. 复检恢复
7. 关闭故障

## 3. 当前开发阶段
当前只推进：
- 工程骨架
- 开放平台最小联通性验证
- 点位同步最小链路（分组 + 分组下设备）

当前不做：
- 巡检判帧
- 工单流转实现
- 企业微信通知实现
- AI 识别
- 大屏驾驶舱
- 复杂报表
- 多渠道通知

## 4. 首版必须收敛的页面
- 运行总览
- 点位态势地图
- 视频巡检
- 异常处理中心
- 工单处理
- 点位详情
- 基础配置
- 主题设置

当前阶段只允许先做占位页或最小只读能力，不提前扩展完整业务流。

## 5. 点位详情页原则
点位详情页是单点运维处置页，不是普通资料页。
进入后应快速回答：
- 当前状态是什么
- 异常是什么
- 证据是什么
- 谁在处理
- 当前工单到哪一步
- 还能执行什么动作

平台同步信息与用户人工维护信息必须分离。

## 6. 工单与异常边界（后续轮次使用）
### 固定异常类型
- 设备离线
- 在线但无法播放
- 在线但黑屏
- 在线但画面异常

### 固定工单状态
- 待派单
- 已派单
- 处理中
- 待复核
- 已恢复
- 已关闭

## 7. 地图与主题约束
- 地图必须使用高德地图 JS API
- 坐标转换必须使用高德地图 JS 接口，不使用 webservice 接口
- 首版主题支持：System / Light / Dark
- 基于 .NET 10 WPF 与 Fluent 风格主题能力实现

## 8. 日志与调试约束
- 日志文件统一放项目根目录
- 每次运行前删除旧日志文件
- 必须保留完整 Debug 日志和完整 API 访问日志
- 调试信息不在 UI 展示
- 原始运行日志不得直接交给 Codex
- 日志先由人工提炼结论，再形成下一轮提示词

## 9. Codex 协作约束
- 每一轮任务必须新开线程
- 每轮只做一个明确范围的小任务
- 不得在旧线程上继续堆叠上下文
- 不得读取仓库外 skill / 模板 / 隐藏背景
- 每轮完成后必须：
  - git add
  - git commit
  - git push origin main
- 只允许在 main 分支工作，不得切换分支

## 10. 当前建议推进顺序
1. 工程骨架
2. 开放平台最小联通性验证（getAccessToken）
3. 分组与分组下设备同步
4. 运行总览最小统计
5. 地图真实点位渲染
6. 单点巡检最小链路
7. 异常入池
8. 工单与通知
