# 天翼视联点位巡检系统项目基线（当前有效版）

## 1. 文件定位

本文档用于约束当前项目在仓库内已经真实落地、可继续复用的开发基线。
后续页面、字段、代码结构、Codex 提示词、开发顺序和验收边界，均应以本文档为准。

## 2. 当前阶段

当前阶段为：

**Phase 0 / 首版最小闭环收口**

当前轮次目标已经收口到两个结果：

1. 异常池中的单条异常支持最小复检与恢复确认。
2. 真实设备目录与所有点位已经按“全量目录 / 全量设备”口径落地到本地 SQLite，并可在应用 UI 中按目录总数、设备总数和分组明细直接人工核对。

## 3. 当前已完成状态

截至当前版本，以下能力已经在 `main` 真实落地：

### 3.1 平台接入与安全基线

- 开放平台鉴权链路可用
- 请求公共参数继续按平台协议组装
- 请求 `params` 固定使用 XXTea
- 响应固定只解 `data`
- 当前 `version=1.1` 响应 `data` 固定使用 RSA 私钥解密
- RTSP 预览链路继续固定使用 `rtp over tcp`
- 日志继续对 token、密钥、RTSP 地址做脱敏

### 3.2 本地同步与基础展示

- 分组同步：`getGroupList`
- 分组下设备同步：`getGroupDeviceList`
- 同步口径已收口为：拉取平台返回的全部分组，以及每个分组下返回的全部设备
- 分组与设备已持久化到本地 SQLite
- 分组/设备同步失败时，为避免局部落地，本地 SQLite 保留上一版完整快照
- 运行总览最小统计可用
- 地图页真实点位渲染可用
- 地图页中文化与未定位列表可用

### 3.3 单点预览与单点巡检

- 单点预览页可从本地 SQLite 选择点位
- 可查询设备状态
- 可请求 RTSP 地址
- 可打开独立播放窗口
- 单点实际 RTSP 播放最小链路已打通
- 单点最小巡检诊断可复用现有状态查询、RTSP 获取和播放探测链路

### 3.4 异常池最小收口

- 当前对外前置归类文本固定为：
  - 离线
  - RTSP 未就绪
  - 播放失败
  - 无异常/巡检通过
- 异常池使用本地 SQLite 持久化
- 最小去重口径固定为 `DeviceCode + AbnormalClass + Conclusion`
- 已复核状态可持久化并在重启后恢复
- 处置状态可持久化并在重启后恢复
- 处置状态最小收口仍为：
  - 待处理
  - 处理中
  - 已处理

### 3.5 本轮新增落地

- 异常池单条异常支持最小“复检”入口
- 复检继续复用现有单点巡检链路，不新增第二套巡检链路
- 复检仍异常时，更新原异常记录的最近巡检时间、最新结论与最新摘要，不新增重复记录
- 复检通过时，在原异常记录上最小标记“已恢复确认”
- 恢复确认字段已落到现有异常池表，不新增第二张恢复表
- 真实设备目录与所有点位已在预览页内按全量口径可见，不再只停留在 SQLite 落地或统计数
- 预览页内显式展示目录总数、设备总数、本地 SQLite 快照计数和平台分组设备数汇总
- 预览页内保留空目录，并在分组级展示设备计数，便于人工核对是否完整
- 分组与设备目录读取统一走本地 SQLite 全量设备口径，避免 UI 只显示局部结果

## 4. 当前固定业务主线

当前固定业务主线仍为：

**点位同步 -> 视频巡检 -> 异常归类 -> 故障入池 -> 复检恢复 -> 关闭故障**

本阶段仍未进入完整工单、通知和批量调度。

## 5. 当前固定边界

- 自动调度

## 6. 当前应用内最小 UI 口径

当前应用内已经可以直接查看：

- 真实分组目录
- 分组下全部点位/设备
- 目录总数
- 设备总数
- 本地 SQLite 快照计数
- 平台分组设备数汇总
- 设备名称
- 设备编码
- 如现有数据模型中已有状态，则最小状态文本
- 异常池中的已复核状态
- 异常池中的处置状态
- 异常池中的是否已恢复确认
- 单条异常上的复检入口

## 7. 当前日志口径

日志继续保留：

- API 调用日志
- 应用运行日志
- 全量目录同步开始日志
- 每个分组的设备拉取结果日志
- 全量同步汇总日志
- 单点巡检日志
- 异常入池日志
- 异常复检开始/完成日志
- 复检后命中更新 / 恢复确认日志
- 真实目录与点位加载、全量目录/设备绑定结果日志
