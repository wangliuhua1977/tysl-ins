# 天翼视联点位巡检系统项目基线（当前有效版）

## 1. 文件定位

本文档用于约束当前仓库内已经真实落地、可继续复用的项目基线。后续页面、字段、代码结构、开发顺序和验收边界，均应以本文档为准。

## 2. 当前阶段

当前阶段为：

**Phase 0 / 首版最小闭环收口**

当前基线已经收口到三条已完成主线：

1. 单点预览、单点视频巡检、异常池最小去重、单条复检与恢复确认。
2. 真实监控目录树与目录层级设备的全量同步、全量落地、全量读取与 UI 展示。
3. 点位详情最小深化，以及平台百度坐标转高德坐标后的统一地图渲染口径。

## 3. 当前已完成状态

截至当前版本，以下能力已经在 `main` 真实落地：

### 3.1 平台接入与安全基线

- 开放平台鉴权链路可用。
- 请求公共参数继续按平台协议组装。
- 请求 `params` 固定使用 XXTea。
- 响应固定只解 `data` 字段。
- 当前 `version=1.1` 时，响应 `data` 固定使用 RSA 私钥解密。
- RTSP 预览与播放链路继续固定使用 `rtp over tcp`。
- 日志继续对 token、密钥、签名、Webhook、RTSP 地址做脱敏。

### 3.2 监控目录树全量同步主链路

- 目录同步主链路已经切换到监控目录树接口组合，不再以 `getGroupList / getGroupDeviceList` 作为“全量目录 / 全量设备”主链路。
- 首层目录从 `getReginWithGroupList(regionId="")` 开始拉取。
- 目录树按照 `hasChildren=1` 递归拉全。
- 目录层级设备按照 `havDevice=1` 调用 `getDeviceList(regionId, pageNo, pageSize=50)` 分页拉全。
- 空目录会被保留，不会因为无设备而被吞掉。
- 设备同步不会因为无坐标、离线、无云存、无法预览而被从“全量目录设备”口径里丢掉。
- SQLite 快照改为事务性整体替换；任一目录树或目录设备拉取失败时，保留上一版完整快照，避免出现“半新半旧”的误导性本地数据。
- `getCusDeviceCount` 已接入首层 `regionCode` 的最小对账能力。
- 当前首层 `regionCode` 对账差异已经从“未知差异”收口到“已解释差异 / 一致结果”口径，不再以“未知差异待查”作为当前主线。

### 3.3 本地落地与展示

- 监控目录树节点和目录层级设备已完整落地到本地 SQLite。
- 预览页、地图页与配置页已经统一从本地 SQLite 快照读取目录和设备，不再混用旧的自定义分组口径。
- UI 中已经可以直接区分目录节点与设备节点。
- UI 中已经可以直接查看目录总数、设备总数、SQLite 快照目录数 / 设备数和最小一致性提示。
- 运行总览中的总点位统计已与当前目录设备快照口径对齐。
- 地图页真实点位渲染、中文化与未定位列表仍可用。

### 3.4 单点预览、播放与最小巡检

- 可从本地 SQLite 选择真实设备点位。
- 可查询设备在线状态。
- 可请求 RTSP 地址。
- 可打开独立播放窗口。
- 单点实际 RTSP 播放最小链路已打通。
- 单点最小视频巡检诊断可复用当前状态查询、RTSP 获取与播放探测链路。

### 3.5 异常池最小收口

- 当前前置归类仍固定为：
  - 离线
  - RTSP 未就绪
  - 播放失败
  - 无异常 / 巡检通过
- 异常池使用本地 SQLite 持久化。
- 最小去重口径固定为 `DeviceCode + AbnormalClass + Conclusion`。
- 已复核状态、处置状态、恢复确认状态均可持久化并在重启后恢复。
- 单条异常支持复检入口。
- 复检仍异常时更新原记录，不新增重复异常。
- 复检通过时在原记录上最小标记“已恢复确认”。

### 3.6 点位详情与地图坐标口径

- 预览页点位详情已明确拆分为“平台设备信息”和“用户维护信息”两个区域。
- 平台设备信息区当前至少展示：
  - 设备名称
  - 设备编码
  - 所属目录路径
  - 在线状态
  - 平台原始经纬度
  - 地址 / 位置编码（若当前已有）
  - 最近巡检结论
  - 异常池摘要
  - 处置状态
  - 恢复确认状态
- 用户维护信息区当前已支持本地 SQLite 最小持久化，字段为：
  - 维护状态
  - 维护备注
  - 最近人工确认说明
  - 最近保存时间
- 选中单个点位或地图需要坐标时，当前会按 `deviceCode` 调用 `getDeviceInfoByDeviceCode` 补齐平台原始坐标，不再把 `showDevice` 当作坐标主来源。
- 地图页当前固定把平台原始坐标按百度坐标处理，并在渲染前统一转换成高德坐标。
- marker 只允许使用转换后的高德坐标；坐标转换失败时会明确标识“坐标转换失败，需人工确认”，不会直接按原始坐标绘制。
- 预览页详情与地图页信息面板当前都会明确区分：
  - 平台原始坐标
  - 坐标来源（平台 / 无）
  - 地图渲染坐标（高德）
  - 坐标状态

## 4. 当前固定业务主线

当前固定业务主线仍为：

**点位同步 -> 视频巡检 -> 异常归类 -> 故障入池 -> 复检恢复 -> 关闭故障**

当前仍未进入完整工单、通知和批量巡检阶段。

- 每个目录下的目录层级设备
- 空目录
- 目录总数
- 设备总数
- SQLite 快照目录数 / 设备数
- 首层 `getCusDeviceCount` 最小对账提示
- 设备名称
- 设备编码
- 单点预览与单点巡检入口
- 结构化点位详情
- 用户维护信息最小保存入口

- 异常池中的已复核、处置状态、恢复确认状态与复检入口

日志继续保留：

- API 调用日志
- 应用运行日志
- 监控目录树递归开始 / 完成日志
- 单个目录设备分页拉取结果日志
- 全量同步汇总日志
- SQLite 快照替换结果日志
- UI 目录 / 设备绑定结果日志
- `getCusDeviceCount` 最小对账结果日志

- 百度坐标转高德坐标开始 / 完成 / 失败日志

日志仍不得输出完整 RTSP 明文。

## 8. 当前风险与未完成项

- `getDeviceList` 文档最小只保证 `deviceCode / deviceName`，当前目录设备同步不会把它当成经纬度、在线状态、云存状态等完整详情来源。
- 当前坐标补齐只覆盖“按 `deviceCode` 拉平台基础信息并落本地 SQLite”的最小链路；尚未扩展到 `showDevice` 详情补充、用户补录坐标或批量坐标修正。
- 预览页与地图页当前都可看到平台原始坐标和高德渲染坐标，但地图绘制仍只允许使用转换成功后的高德坐标。
- 用户维护信息当前仅覆盖最小字段，尚未进入责任映射、通知映射、CSV 批量导入导出或坐标修正闭环。
- 坐标转换失败的点位当前会明确提示人工确认，但尚未提供用户补录坐标入口。

## 9. 当前推荐下一步

当前最合理的下一步最小任务为：

**用户补录坐标最小闭环**

建议只做最小承接：

1. 为点位增加用户补录坐标的最小持久化与录入口径。
2. 在地图页与点位详情中落实“用户补录坐标 > 平台坐标 > 未定位”的固定优先级。
3. 保持当前开放平台安全基线、RTSP 链路、地图接入方式和最小详情结构不变。

## 10. 当前一句话项目状态

**当前项目已经完成“开放平台安全基线 + 监控目录树全量递归同步与分页设备拉取 + SQLite 事务性快照落地 + 运行总览最小统计 + 单点预览 / 播放 / 巡检最小链路 + 异常池最小去重、复核、处置状态与恢复确认 + 结构化点位详情 + 用户维护信息最小持久化 + 地图页平台百度坐标转高德坐标后的统一渲染”，下一步应优先补齐用户坐标修正最小闭环，而不是扩展工单、通知或复杂详情系统。**
