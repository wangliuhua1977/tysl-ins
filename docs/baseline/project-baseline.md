# 天翼视联点位巡检系统项目基线（当前有效版）

## 1. 文件定位

本文档用于约束当前仓库内已经真实落地、可继续复用的项目基线。后续页面、字段、代码结构、提示词、开发顺序和验收边界，均应以本文档为准。

## 2. 当前阶段

当前阶段为：

**Phase 0 / 首版最小闭环收口**

当前基线已经收口到两条已完成主线：

1. 单点预览、单点视频巡检、异常池最小去重、单条复检与恢复确认。
2. 真实监控目录树与目录层级设备的全量同步、全量落地、全量读取与 UI 展示。

## 3. 当前已完成状态

截至当前版本，以下能力已经在 `main` 真实落地：

### 3.1 平台接入与安全基线

- 开放平台鉴权链路可用。
- 请求公共参数继续按平台协议组装。
- 请求 `params` 固定使用 XXTea。
- 响应固定只解 `data` 字段。
- 当前 `version=1.1` 时，响应 `data` 固定使用 RSA 私钥解密。
- RTSP 预览与播放链路继续固定使用 `rtp over tcp`。
- 日志继续对 token、密钥、签名、Webhook、RTSP 地址做脱敏。

### 3.2 监控目录树全量同步主链路

- 目录同步主链路已经切换到监控目录树接口组合，不再以 `getGroupList / getGroupDeviceList` 作为“全量目录 / 全量设备”主链路。
- 首层目录从 `getReginWithGroupList(regionId="")` 开始拉取。
- 目录树按照 `hasChildren=1` 递归拉全。
- 目录层级设备按照 `havDevice=1` 调用 `getDeviceList(regionId, pageNo, pageSize=50)` 分页拉全。
- 空目录会被保留，不会因为无设备而被吞掉。
- 设备同步不会因为无坐标、离线、无云存、无法预览而被从“全量目录设备”口径里丢掉。
- SQLite 快照改为事务性整体替换；任一目录树或目录设备拉取失败时，保留上一版完整快照，避免出现“半新半旧”的误导性本地数据。
- `getCusDeviceCount` 已接入首层 `regionCode` 的最小对账能力。

### 3.3 本地落地与展示

- 监控目录树节点和目录层级设备已完整落地到本地 SQLite。
- 预览页与配置页已经统一从本地 SQLite 快照读取目录和设备，不再混用旧的自定义分组口径。
- UI 中已经可以直接区分目录节点与设备节点。
- UI 中已经可以直接查看目录总数、设备总数、SQLite 快照目录数/设备数和最小一致性提示。
- 运行总览中的总点位统计已与当前目录设备快照口径对齐。
- 地图页真实点位渲染、中文化与未定位列表仍可用。

### 3.4 单点预览、播放与最小巡检

- 可从本地 SQLite 选择真实设备点位。
- 可查询设备在线状态。
- 可请求 RTSP 地址。
- 可打开独立播放窗口。
- 单点实际 RTSP 播放最小链路已打通。
- 单点最小视频巡检诊断可复用当前状态查询、RTSP 获取与播放探测链路。

### 3.5 异常池最小收口

- 当前前置归类仍固定为：
  - 离线
  - RTSP 未就绪
  - 播放失败
  - 无异常 / 巡检通过
- 异常池使用本地 SQLite 持久化。
- 最小去重口径固定为 `DeviceCode + AbnormalClass + Conclusion`。
- 已复核状态、处置状态、恢复确认状态均可持久化并在重启后恢复。
- 单条异常支持复检入口。
- 复检仍异常时更新原记录，不新增重复异常。
- 复检通过时在原记录上最小标记“已恢复确认”。

## 4. 当前固定业务主线

当前固定业务主线仍为：

**点位同步 -> 视频巡检 -> 异常归类 -> 故障入池 -> 复检恢复 -> 关闭故障**

当前仍未进入完整工单、通知和批量巡检阶段。

## 8. 当前风险与未完成项

- `getCusDeviceCount` 目前只完成首层 `regionCode` 对账，不代表整棵目录树已逐节点完成对账。
- `getDeviceList` 文档最小只保证 `deviceCode / deviceName`，当前目录设备同步不会把它当成经纬度、在线状态、云存状态等完整详情来源。
- 2026-03-29 真实 `--sync-once` 样例结果为：`directories=10 devices=19 snapshotReplaced=True reconcileCompleted=True reconcileMatched=False`。
- 当前已识别的首层对账差异为：
  - `交通要道(94943605-4)` 缺少平台对账记录
  - `我的设备(94943605-6)` 本地 `7` / 平台 `5`
- 上述差异仍需后续人工确认或下一轮最小收口，当前不得写成“已确认完整”。
