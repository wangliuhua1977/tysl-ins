# 单点预览、异常复检与真实监控目录树说明（当前有效版）

## 1. 文件定位

本文档用于约束当前项目中与“单点预览 + 独立播放窗口 + 最小巡检诊断 + 异常池复检恢复 + 真实监控目录树展示”相关的真实链路。本文档只描述已经落地并允许继续复用的实现，不描述未来扩展方案。

## 2. 当前已完成范围

截至当前版本，以下能力已经真实落地：

1. 预览页从本地 SQLite 读取真实设备点位。
2. 预览页中的目录来源已经切换到真实监控目录树快照，不再把自定义分组当成“全量目录 / 全量设备”口径。
3. 预览页可以显示目录、目录层级设备、空目录、目录总数、设备总数、SQLite 快照目录数 / 设备数和最小一致性提示。
4. 单点状态诊断链路可用。
5. RTSP 地址获取链路可用。
6. 独立播放窗口链路可用。
7. 播放链路固定使用 `rtp over tcp`。
8. 单点最小巡检诊断链路可用。
9. 异常结果可进入本地异常池并执行最小去重。
10. 异常池条目支持已复核、处置状态、单条复检与恢复确认。

## 3. 当前固定链路

当前固定链路如下：

1. 同步阶段先把真实监控目录树和目录层级设备落地到本地 SQLite。
2. 预览页从本地快照绑定目录树和设备。
3. 用户从目录树或设备列表中选中单个设备。
4. 先查询设备状态。
5. 状态允许时获取 RTSP 地址。
6. RTSP 地址就绪后可打开独立播放窗口。
7. 用户可发起一次最小巡检诊断。
8. 诊断异常进入本地异常池。
9. 用户可在异常池条目上点击“复检”。
10. 复检仍异常时更新原记录，不新增重复记录。
11. 复检通过时在原记录上标记“已恢复确认”。

## 4. 当前 UI 口径

预览页当前至少展示以下信息：

### 4.1 真实监控目录树区域

- 目录名称
- 目录层级 / 父子关系
- 目录设备数
- 空目录提示
- 设备名称
- 设备编码
- 目录总数
- 设备总数
- SQLite 快照目录数 / 设备数
- 首层 `getCusDeviceCount` 最小对账提示

### 4.2 单点预览区域

- 当前设备名称
- 当前设备编码
- 当前诊断状态
- 地址获取结果
- RTSP 地址
- 有效期 / 失效时间
- 最近一次请求时间
- 获取预览地址按钮
- 巡检诊断按钮
- 打开播放窗口按钮

### 4.3 异常池区域

- 设备名称
- 最近巡检时间
- 前置归类
- 结论
- 已复核
- 处置状态
- 是否已恢复确认
- 恢复确认时间
- 恢复说明
- 复检操作

## 5. 当前与监控目录树主链路的关系

- 预览页设备来源依赖于监控目录树全量同步结果。
- 目录同步主链路固定为：
  - `getReginWithGroupList`
  - `getDeviceList`
  - 最小对账：`getCusDeviceCount`
- 如果目录树同步任一关键步骤失败，本地继续保留上一版完整快照，预览页不会展示“半新半旧”的误导性目录数据。
- 当前 `getDeviceList` 文档最小只保证 `deviceCode / deviceName`，因此预览页不会把该接口直接当成经纬度、在线状态或云存状态的完整详情来源。

## 6. 当前日志口径

当前必须记录：

- 监控目录树同步开始 / 完成
- 单个目录设备分页拉取结果
- 全量同步汇总
- SQLite 快照替换结果
- 真实目录 / 设备加载与 UI 绑定结果
- 首层 `getCusDeviceCount` 最小对账结果
- 单点预览准备开始 / 完成
- 单点巡检开始 / 完成
- 异常入池
- 异常去重更新
- 异常复检开始 / 完成
- 异常复检后命中更新
- 异常复检后恢复确认

日志仍需满足：

- 不输出完整 RTSP 地址
- 对 token、密钥、Webhook 等敏感值继续脱敏

## 7. 当前明确不做

当前文档范围内仍然不做：

- 批量巡检
- 自动调度
- 自动刷新 RTSP 地址
- 自动截图
- 黑屏判断
- 工单
- 通知
- 多点同时播放
