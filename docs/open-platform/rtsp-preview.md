# 单点预览与独立播放窗口说明（当前有效版）

## 1. 文件定位

本文件用于约束当前项目中“单点预览 + 独立播放窗口 + 最小巡检诊断 + 最小异常池”相关的真实链路。

本文件只描述当前已经落地并允许继续复用的链路，不描述未来推演方案。

当前相关能力已经真实落地到 `main`：

- 单点状态诊断
- RTSP 地址获取
- 独立播放窗口打开
- LibVLC 实际建链与播放状态回调
- 单点最小巡检诊断
- 最小异常池持久化、最小去重与已复核恢复

后续任何与单点播放、视频巡检、最小诊断轮相关的开发，都必须先以本文件为准。

---

## 2. 当前已完成边界

截至当前版本，以下内容已完成并允许复用：

1. 预览页从本地 SQLite 选择单个点位
2. 调用设备状态接口做预览前诊断
3. 调用 RTSP 接口获取直播地址
4. 仅在 RTSP 地址已就绪时允许打开独立播放窗口
5. 独立播放窗口内使用 VLC 播放 RTSP
6. 播放链路固定使用 `rtp over tcp`
7. 播放状态以中文口径反馈到窗口 UI
8. 播放日志保留掩码 RTSP，不明文输出完整地址
9. 单点预览页可直接发起一次“巡检诊断”
10. 巡检诊断复用现有状态查询、RTSP 获取、VLC 播放链路，输出最小诊断结果
11. 已在单点预览页补齐“前置归类结果”最小展示，归类只收口为：
   - 无异常/巡检通过
   - 离线
   - RTSP 未就绪
   - 播放失败
12. 已在单点预览页补齐“最小处置摘要”展示：
   - 复用当前 `InspectResult`
   - 输出巡检时间、设备名称、设备编码、巡检结论、前置归类、失败分类（如有）与最小诊断信息
   - 摘要为中文短文本，可直接人工复制
   - 摘要不展示 RTSP 地址明文
13. 已在单点预览页补齐“当前异常池列表”最小链路：
   - 仅“离线 / RTSP 未就绪 / 播放失败”进入异常池
   - 异常池使用本地 SQLite `InspectAbnormalPool` 表持久化
   - 最小去重键固定为 `DeviceCode + AbnormalClass + Conclusion`
   - 命中去重时不新增第二条，只更新最近巡检时间、最新摘要与 `UpdatedAt`，并保留已复核状态
   - 列表展示设备、巡检时间、前置归类、结论、最小处置摘要与已复核状态
   - 支持最小“已复核”标记切换并持久化
   - 应用重启后可恢复显示已持久化异常项

当前固定播放方案：

- `LibVLCSharp.WPF 3.9.6`
- `VideoLAN.LibVLC.Windows 3.0.23`
- 独立播放窗口
- 不嵌入主 Tab

## 5. 固定链路说明

### 5.1 预览页职责

预览页只负责：

- 选择点位
- 发起状态诊断
- 获取 RTSP 地址
- 发起一次最小巡检诊断
- 展示中文诊断结果
- 将异常结果写入当前异常池
- 切换异常项“已复核”标记
- 在地址就绪后允许打开独立播放窗口

预览页不负责：

- 播放器初始化
- 播放状态事件处理
- 二次鉴权
- 再次取流

### 5.2 播放窗口职责

独立播放窗口只负责：

- 接收现成的设备名称、设备编码、RTSP 地址
- 初始化 VLC
- 建立 RTSP 播放链路
- 回传最小播放状态
- 停止播放
- 关闭窗口时释放资源

播放窗口不得：

- 重新获取 accessToken
- 重新请求 RTSP 地址
- 修改开放平台接口调用规则

### 5.3 当前固定调用方式

当前固定链路如下：

1. 预览页选中设备
2. 先查设备状态
3. 状态允许时获取 RTSP 地址
4. RTSP 地址就绪后点击“打开播放窗口”
5. `PlayWinSvc` 打开独立窗口
6. `PlayWinViewModel` 将现成 RTSP 传给 `VlcPlaySvc`
7. `VlcPlaySvc.Initialize/Start/Stop/Dispose`
8. VLC 事件驱动窗口状态更新
9. 单点巡检异常结果进入本地持久化异常池

## 6. 接口一：设备状态查询

### 6.1 接口信息

- 接口名：`getDeviceStatus`
- 路径：`/open/token/vpaas/device/getDeviceStatus`
- 完整地址：`https://vcp.21cn.com/open/token/vpaas/device/getDeviceStatus`

### 6.2 当前最小请求参数

- `accessToken`
- `enterpriseUser`
- `deviceCode`

### 6.3 当前关键返回字段

`data` 解密后至少关注：

- `status`
- `deviceCode`

### 6.4 当前状态口径

- `1`：在线
- `0`：离线
- `2`：休眠（普通休眠）
- `3`：休眠（保活）

### 6.5 当前中文诊断口径

- `1`：在线，可请求预览地址
- `0`：设备离线，无法获取预览地址
- `2`：设备休眠，当前不进入预览
- `3`：设备休眠，当前不进入预览
- 其他：设备状态未知，当前不进入预览

---

## 7. 接口二：获取 RTSP 直播地址

### 7.1 接口信息

- 接口名：`getDeviceMediaUrlRtsp`
- 路径：`/open/token/cloud/getDeviceMediaUrlRtsp`
- 完整地址：`https://vcp.21cn.com/open/token/cloud/getDeviceMediaUrlRtsp`

### 7.2 当前最小请求参数

- `accessToken`
- `enterpriseUser`
- `deviceCode`
- `expire`（可选）
- `netType`（可选，默认公网）

### 7.3 当前关键返回字段

`data` 解密后重点关注：

- `url`
- `expireTime`

### 7.4 地址有效期说明

文档口径如下：

- 国标 / 社标设备：5 分钟
- 企标 2.0：10 分钟
- 企标 3.0 / 4.0：15 分钟

当前项目不得假设地址固定有效。

### 7.5 当前固定约束

- RTSP 地址不是固定值
- 同一设备每次获取的地址可能不同
- 地址一段时间无人使用后会失效
- 客户端必须使用 **rtp over tcp**
- 当前播放窗口不做自动重取地址
- 当前播放窗口不做自动重试

## 8. 播放接入固定规则

### 8.1 固定依赖

- `LibVLCSharp.WPF 3.9.6`
- `VideoLAN.LibVLC.Windows 3.0.23`

### 8.2 固定建链方式

当前实现固定使用：

```csharp
new LibVLC("--rtsp-tcp")
```

不允许在当前阶段引入以下替代方案：

- VLC 外部 EXE 拉起方案
- FFmpeg 方案
- 多种 RTSP 试错策略混用
- 主 Tab 内嵌播放器重构

### 8.3 固定状态分类

正常状态：

- `正在初始化播放器`
- `正在建立播放链路`
- `正在播放`
- `已停止播放`

失败分类：

- `播放初始化失败`
- `播放建链失败`
- `播放过程中断`
- `地址可能失效`

### 8.4 状态边界

播放器失败不得回写成接口失败。

接口侧失败分类继续保持独立：

- `RTSP 接口业务失败`
- `RTSP 响应解密失败`
- `RTSP 解密后 JSON 解析失败`
- `RTSP 返回缺少 url`

## 12. 当前阶段验收口径

本文件相关实现通过的标准是：

1. 能从本地点位中选中设备
2. 能完成一次状态诊断
3. 满足条件时能获取 RTSP 地址
4. 仅在地址就绪时允许打开播放窗口
5. 独立窗口能使用 VLC 发起 RTSP 播放
6. 播放状态能给出中文结果
7. 失败时能明确归类到接口侧或播放器侧
8. 日志完整且脱敏
9. 能输出最小巡检诊断结果
10. 能输出“离线 / RTSP 未就绪 / 播放失败 / 无异常”前置归类结果
11. 异常结果能进入本地持久化异常池
12. 巡检通过不会进入异常池
13. 相同 `DeviceCode + AbnormalClass + Conclusion` 不会生成第二条记录
14. “已复核”切换后写回 SQLite，应用重启后仍可恢复
15. 不破坏现有地图、同步、总览能力
