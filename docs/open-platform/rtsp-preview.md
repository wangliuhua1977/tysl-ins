# 单点预览、点位详情与地图坐标说明（当前有效版）

## 1. 文件定位

本文档用于约束当前项目中与以下链路直接相关的真实实现：

- 单点预览
- 独立播放窗口
- 最小巡检诊断
- 异常池复检与恢复确认
- 预览页点位详情最小深化
- 地图页平台百度坐标转高德坐标后的渲染口径

本文档只描述仓库内已经真实落地并允许继续复用的实现，不描述未来扩展方案。

## 2. 当前已完成范围

截至当前版本，以下能力已经真实落地：

1. 预览页从本地 SQLite 读取真实监控目录树与目录层级设备。
2. 用户可在同一页内选择真实点位并发起单点预览、单点巡检与异常复检。
3. RTSP 地址获取链路可用，且播放链路继续固定使用 `rtp over tcp`。
4. 异常池最小去重、处置状态、已复核与恢复确认链路可用。
5. 选中点位后，详情区已整理为两个清晰区域：
   - 平台设备信息
   - 用户维护信息
6. 用户维护信息已支持本地 SQLite 最小持久化，当前字段为：
   - 维护状态
   - 维护备注
   - 最近人工确认说明
   - 最近保存时间
7. 地图页在渲染前会把平台原始坐标按“百度坐标”处理，并通过高德 JS API 官方坐标转换能力转换为高德坐标后再绘制。
8. 原始平台坐标与地图渲染坐标已明确区分；坐标转换失败时不会静默按原坐标绘制。

## 3. 当前固定链路

当前固定链路如下：

1. 同步阶段先将真实监控目录树和目录层级设备落地到本地 SQLite。
2. 预览页从本地快照绑定目录树、设备和点位详情基础信息。
3. 用户从目录树、目录设备列表或点位下拉框中选中单个设备。
4. 预览页先加载当前选中点位的最小详情：
   - 平台设备信息
   - 用户维护信息
5. 单点预览前先查询设备在线状态。
6. 在线时再请求 RTSP 地址。
7. RTSP 地址就绪后可打开独立播放窗口。
8. 用户可发起一次最小巡检诊断。
9. 诊断异常时进入本地异常池。
10. 用户可对异常池条目发起复检；复检仍异常则更新原记录，复检通过则在原记录上标记“已恢复确认”。
11. 地图页加载本地点位后，先保留平台原始经纬度，再统一执行“百度 -> 高德”转换；只有转换成功的点位才参与 marker 渲染。

## 4. 当前 UI 口径

### 4.1 真实监控目录树区域

当前至少展示：

- 目录名称
- 目录层级 / 父子关系
- 目录设备数
- 空目录提示
- 设备名称
- 设备编码
- 设备在线状态
- 目录总数
- 设备总数
- SQLite 快照目录数 / 设备数
- 首层 `getCusDeviceCount` 最小对账提示

### 4.2 单点预览区域

当前至少展示：

- 当前设备名称
- 当前设备编码
- 当前诊断状态
- 地址获取结果
- RTSP 地址
- 有效期 / 失效时间
- 最近一次请求时间
- 获取预览地址按钮
- 巡检诊断按钮
- 打开播放窗口按钮

### 4.2.1 点位详情区域

当前点位详情已固定拆分为两个清晰区域。

#### 平台设备信息

当前至少展示：

- 设备名称
- 设备编码
- 所属目录路径
- 在线状态
- 原始坐标来源：百度
- 平台原始经纬度
- 地图渲染口径：高德 JS API 官方转换后使用
- 地址 / 位置编码（若当前已有）
- 最近巡检结论
- 异常池摘要
- 处置状态
- 恢复确认状态

说明：

- 当前详情区展示的平台经纬度仍是平台原始值。
- 地图实际绘制坐标以转换后的高德坐标为准。
- 当前没有稳定来源的字段不会在详情区猜测补齐。

#### 用户维护信息

当前最小维护口径为：

- 维护状态
- 维护备注
- 最近人工确认说明
- 最近保存时间
- 保存维护信息按钮

当前不扩展：

- 责任人流转
- 审批
- 工单
- 通知
- 截止时间
- 图片上传

### 4.3 地图页区域

地图页当前固定口径如下：

- 地图底图继续使用高德地图 JS API。
- 原始平台经纬度按百度坐标处理。
- marker 只允许使用转换后的高德坐标。
- 无坐标点位进入“未定位点位”列表。
- 坐标转换失败的点位进入“未上图点位”列表，并明确标识“坐标转换失败，需人工确认”。
- 点位详情面板可同时看到：
  - 平台原始坐标
  - 地图渲染坐标
  - 坐标状态
  - 坐标说明

### 4.4 异常池区域

当前至少展示：

- 设备名称
- 最近巡检时间
- 前置归类
- 结论
- 已复核
- 处置状态
- 是否已恢复确认
- 恢复确认时间
- 恢复说明
- 复检操作

## 5. 与监控目录树主链路的关系

- 预览页与地图页都继续依赖本地 SQLite 快照，不直接把旧自定义分组当成“全量目录 / 全量设备”口径。
- 当前目录同步主链路继续固定为：
  - `getReginWithGroupList`
  - `getDeviceList`
  - 最小对账：`getCusDeviceCount`
- `getDeviceList` 文档最小只保证 `deviceCode / deviceName`，因此当前详情区不会把该接口直接当成完整设备详情来源。
- 本轮没有改动开放平台鉴权、请求加密、响应解密或 RTSP 获取基线。

## 6. 当前日志口径

当前至少记录：

- 监控目录树加载与 UI 绑定结果
- 点位详情加载开始 / 完成
- 单点预览准备开始 / 完成
- 单点巡检开始 / 完成
- 异常入池
- 异常复检开始 / 完成
- 用户维护信息保存结果
- 百度坐标转高德坐标开始 / 完成 / 失败

日志仍需满足：

- 不输出完整 RTSP 地址
- 对 token、密钥、签名、Webhook 等敏感值继续脱敏

## 7. 当前明确不做

当前范围内仍然不做：

- 批量巡检
- 自动调度
- 自动刷新 RTSP 地址
- 自动截图
- 黑屏判断
- 工单
- 通知
- 多点同时播放
- 复杂详情页导航
- 复杂用户维护表单

## 8. 当前推荐下一步

与本文档直接相关的下一步最小任务，当前优先建议为：

**用户补录坐标最小闭环**

建议只做最小承接：

1. 为点位补充用户坐标录入口径。
2. 在地图与点位详情中落实“用户补录坐标 > 平台坐标 > 未定位”的固定优先级。
3. 继续保持当前 RTSP、安全基线、预览链路与地图接入方式不变。
