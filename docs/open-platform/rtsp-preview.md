# 单点预览、异常复检与真实目录说明（当前有效版）

## 1. 文件定位

本文档用于约束当前项目中与“单点预览 + 独立播放窗口 + 最小巡检诊断 + 异常池复检恢复 + 真实目录展示”相关的真实链路。
本文档只描述已经落地并允许继续复用的实现，不描述未来扩展方案。

## 2. 当前已完成边界

截至当前版本，以下能力已经真实落地：

1. 预览页从本地 SQLite 读取真实点位。
2. 预览页内按“全量目录 / 全量设备”口径展示真实设备目录与所有点位。
3. 目录中可区分分组与设备，并可直接选中点位。
4. 预览页内显式展示目录总数、设备总数、本地 SQLite 快照计数和平台分组设备数汇总。
5. 空目录会继续保留在 UI 中，便于人工核对是否存在遗漏。
6. 全量同步存在失败项时，为避免局部落地，本地 SQLite 保留上一版完整快照。
7. 单点状态诊断链路可用。
8. RTSP 地址获取链路可用。
9. 独立播放窗口链路可用。
10. 播放链路固定使用 `rtp over tcp`。
11. 单点最小巡检诊断链路可用。
12. 异常结果可进入本地异常池并执行最小去重。
13. 异常池条目支持已复核、处置状态和单条复检。
14. 异常池条目复检通过后可最小标记“已恢复确认”。

## 3. 当前固定链路

当前固定链路如下：

1. 预览页加载本地真实分组与设备目录。
2. 用户从目录或下拉框选中单个设备。
3. 先查询设备状态。
4. 状态允许时获取 RTSP 地址。
5. RTSP 地址就绪后可打开独立播放窗口。
6. 用户可发起一次最小巡检诊断。
7. 诊断结果只收口到以下前置归类：
   - 离线
   - RTSP 未就绪
   - 播放失败
   - 无异常/巡检通过
8. 诊断异常进入本地异常池。
9. 用户可在异常池条目上点击“复检”。
10. 复检仍异常时更新原记录，不新增重复记录。
11. 复检通过时在原记录上标记“已恢复确认”。

## 4. 当前预览页 UI 口径

预览页当前至少展示以下信息：

### 4.1 真实目录区

- 分组目录
- 分组下全部点位/设备
- 目录总数
- 设备总数
- 本地 SQLite 快照计数
- 平台分组设备数汇总
- 设备名称
- 设备编码
- 最小状态文本（如本地模型已有）
- 目录选择入口
- 空目录提示

### 4.2 单点预览区

- 当前设备名称
- 当前设备编码
- 当前诊断状态
- 地址获取结果
- RTSP 地址
- 有效期 / 失效时间
- 最近一次请求时间
- 获取预览地址按钮
- 巡检诊断按钮
- 打开播放窗口按钮

### 4.3 巡检结果区

- 诊断结论
- 前置归类结果
- 失败分类
- 阶段状态
- 最小诊断信息
- 巡检时间
- 最小处置摘要

### 4.4 异常池区

- 设备
- 最近巡检时间
- 前置归类
- 结论
- 已复核
- 处置状态
- 是否已恢复确认
- 恢复确认时间
- 恢复说明
- 复检操作

## 5. 当前异常池复检口径

当前复检口径固定如下：

1. 仅对异常池中的单条异常记录提供复检入口。
2. 复检必须复用现有单点巡检链路。
3. 复检仍异常时：
   - 更新原记录的最近巡检时间
   - 更新原记录的最新结论
   - 更新原记录的最新摘要
   - 保留已复核状态
   - 保留处置状态
4. 复检通过时：
   - 不删除原记录
   - 在原记录上标记“已恢复确认”
   - 持久化恢复确认时间与恢复说明
5. “已处理”和“已恢复确认”保持两个独立语义，不扩成新的复杂状态机。

## 6. 当前日志口径

当前必须记录：

- 全量目录同步开始
- 每个分组的设备拉取结果
- 全量同步汇总
- 真实目录/点位加载与 UI 绑定结果
- 单点预览准备开始/完成
- 单点巡检开始/完成
- 异常入池
- 异常去重更新
- 异常复检开始/完成
- 异常复检后命中更新
- 异常复检后恢复确认

日志仍必须满足：

- 不输出完整 RTSP 地址
- 对 token、密钥、Webhook 等敏感值继续脱敏

## 7. 当前明确不做

当前文档范围内仍然不做：

- 批量巡检
- 自动调度
- 自动刷新 RTSP 地址
- 自动截图
- 黑屏判断
- 工单
- 通知
- 多点同时播放

## 8. 当前下一步建议

下一步建议继续保持“小步闭环”，优先进入：

**点位详情页最小信息收口**

仅建议承接：

1. 平台设备信息与用户维护信息双区展示。
2. 与当前目录、预览、异常池做最小跳转或联动。
3. 不扩工单、通知和批量调度。
